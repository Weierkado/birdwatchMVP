相机电量 UI · 设计文档

将 SD 卡容量重包装为相机电量显示，解决玩家概念混淆问题
迭代类型：换皮 + UI 替换 机制变动：无 影响范围：全局文本 + 状态栏 UI

一、背景与动机

现有设计使用"SD 卡"作为局内拍摄资源的容器，但测试反馈显示部分玩家存在以下问题：

  • 不清楚 SD 卡是什么，缺乏直觉认知
  • 误以为游戏目标是把 SD 卡"拍满"，而非选择性地拍高质量照片
  • 产生"要多拍"而非"要精拍"的错误策略倾向
电量是玩家高度熟悉的有限资源概念：用完就没了、无法中途充电、要考虑分配。这与游戏希望传达的"有限机会、谨慎决策"体验高度一致，且不需要额外解释。

本次迭代为纯换皮，不改变任何数值或机制。后续可根据测试结果决定是否引入"不同行为状态消耗不同电量"等机制层迭代。

三、电池 UI 规格

3.1 组件构成

电量显示由两个元素并排组成：

  • 电池造型容器：矩形主体 + 右侧凸起电极头，内部为深色底，填充条从左向右表示剩余电量，随消耗从右向左收缩
  • 百分比数字：显示在电池右侧，格式为 XX%（取整，不显示小数）

3.2 百分比计算

// 剩余电量百分比
pct = Math.round((maxPhotos - photos.length) / maxPhotos * 100)
// 开局 100%，每拍一张约 -6.7%（MAX_PHOTOS=15 时），归 0% 时电量耗尽

3.3 颜色阈值

绿色 — 电量 > 30%
黄色 — 电量 11%–30%
红色 — 电量 1%–10%，填充条与数字同步闪烁
空壳 — 电量 0%，数字显示 0%，电池为空壳
闪烁仅在红色阈值（≤ 10%）触发，避免长时间闪烁干扰注意力。0% 时停止闪烁，显示空电池静态状态即可，玩家此时已无法拍摄。

五、未来机制扩展（备忘，本次不实现）

  • 差异化电量消耗：REMARKABLE 拍摄消耗更多电量，体现"精彩时机的代价"
  • 举起相机即消耗：进入对焦阶段扣少量电量，即使转移未拍成也有代价
  • 备用电池道具：局内可拾取/局外可升级，提供额外容量
