一、设计哲学

焦内序列是进入对焦阶段时动态生成的一条行为状态时间轴，对玩家不可见，但其结果（摄影时机 UI 上的 badge 状态切换）完全可被观察和学习。

三条核心原则

  • 可学习性：序列不是随机时段触发的黑盒，而是有结构的弧线。玩家多次遭遇同一鸟种后，能感知到"这只鸟的对焦节奏"——REMARKABLE 通常在什么位置出现、窗口大概多长。这是技巧深度的来源。
  • 外部状态奖励：在高价值外部状态（INTERESTING / REMARKABLE）时举起相机，焦内序列的质量也更高。捕捉稀有时机的奖励是即时且可感知的，不只是卡牌概率的数字变化。
  • 转移作为收尾信号：转移事件只在 NORMAL 状态之后触发。玩家逐渐学会读懂 badge 回到 NORMAL 这个信号——"窗口快关了"。这是 NORMAL 状态的双重含义：低价值 + 临近结束。
焦内序列的行为弧线是鸟类性格在"操作层"的第二次表达。第一次是移动 pattern（追焦手感），第二次是序列节奏（状态切换规律）。翠鸟：移动快 + 序列爆发短暂；苍鹭：移动缓 + 序列沉稳绵长。两层叠加，鸟的"个性"在手感上真实可感知。

2.2 各阶段说明

阶段说明配置项
安全区间 序列播放期间，安全区间内不会到达转移节点。确保玩家有最低限度的操作窗口，不会入场就立刻转移。 safetyMs
起始状态 序列第一个状态与外部行为状态一致。在 INTERESTING 时举起相机，序列从 INTERESTING 开始；在 REMARKABLE 时举起，从 REMARKABLE 开始。 由外部 behaviorState 决定
中间段 由生成算法按权重和跳转规则填充，段数和每段时长在范围内随机。体现鸟种性格差异。 segmentCount / stateWeights / stateDurations
收尾 NORMAL 序列最后一个有意义的状态段固定为 NORMAL（"窗口即将关闭"信号）。若生成的末段本已是 NORMAL，则直接使用，不额外追加。 硬规则，不可配置
转移 序列末尾标记，触发对焦阶段的转移事件，badge 飞出，进入 REPOSITION 或 LOST。 硬规则，不可配置

三、状态跳转规则

每个状态段结束后，下一个状态由权重抽取，但可选范围受 allowJump 参数约束：

相邻跳转模式（allowJump: false）
寻常 → 寻常 ✓
寻常 → 有趣 ✓
寻常 → 精彩 ✗ 禁止跨级
精彩 → 寻常 ✗ 禁止跨级

适合：节奏舒缓、行为有层次感的鸟种（苍鹭、乌鸫）

自由跳转模式（allowJump: true）
寻常 → 精彩 ✓ 允许跨级
精彩 → 寻常 ✓ 允许跨级
寻常 → 有趣 ✓
有趣 → 精彩 ✓

适合：爆发性强、行为不可预测的鸟种（翠鸟、麻雀）

allowJump 是鸟类个性在序列层的表达。苍鹭不会"突然精彩"，翠鸟可以直接从安静跳入俯冲。玩家在多次遭遇后形成的心理预期不同，前者可预判、后者要反应。两种体验都是合理的，只要在同一鸟种内保持一致。

四、配置数据结构

在现有 focusConfig.js 每个状态配置中新增 sequence 字段：

// focusConfig.js — 每个 [speciesId][behaviorState] 新增 sequence 字段 { pattern: "sweep", // 现有：移动 pattern layers: [ ... ], // 现有：Lissajous 参数 focus: { ... }, // 现有：判定区大小 sequence: { // 新增：焦内序列参数 safetyMs: 1200, // 安全区间（ms），此段内序列不到达转移节点 segmentCount: { min: 2, max: 4 }, // 中间段数量（不含起始段和收尾 NORMAL） allowJump: true, // 是否允许跨级跳转（false = 只能相邻跳转） stateWeights: { // 生成中间段时各状态的抽取权重 NORMAL: 20, INTERESTING: 45, REMARKABLE: 35 }, stateDurations: { // 每种状态的持续时长范围（ms） NORMAL: { min: 500, max: 900 }, INTERESTING: { min: 400, max: 700 }, REMARKABLE: { min: 300, max: 550 } } } }
stateWeights 用于中间段生成，不影响起始状态（起始状态固定跟随外部行为状态）。收尾 NORMAL 由生成算法强制追加，权重中的 NORMAL 比例影响中间段出现 NORMAL 的频率，但不影响收尾逻辑。

五、生成算法

/** * 生成焦内序列 * @param speciesId 鸟种 id * @param outerBehaviorState 外部行为序列的当前状态（决定起始段状态） * @returns FocusSequence { segments: [{state, durationMs}], safetyMs } */ function generateFocusSequence(speciesId, outerBehaviorState) { const cfg = getFocusConfig(speciesId, outerBehaviorState).sequence; const segments = []; // 1. 起始段：固定为外部状态 segments.push({ state: outerBehaviorState, durationMs: randomBetween( cfg.stateDurations[outerBehaviorState].min, cfg.stateDurations[outerBehaviorState].max ) }); // 2. 中间段：按 segmentCount 和 stateWeights 生成 const middleCount = randomBetween(cfg.segmentCount.min, cfg.segmentCount.max); let prevState = outerBehaviorState; for (let i = 0; i < middleCount; i++) { const nextState = pickNextState(prevState, cfg); segments.push({ state: nextState, durationMs: randomBetween( cfg.stateDurations[nextState].min, cfg.stateDurations[nextState].max ) }); prevState = nextState; } // 3. 收尾 NORMAL：若末段已是 NORMAL 则直接使用，否则追加 if (segments[segments.length - 1].state !== "NORMAL") { segments.push({ state: "NORMAL", durationMs: randomBetween( cfg.stateDurations.NORMAL.min, cfg.stateDurations.NORMAL.max ) }); } // 4. 转移标记 segments.push({ state: "TRANSFER", durationMs: 0 }); return { segments, safetyMs: cfg.safetyMs }; } // 跳转规则：allowJump=false 时过滤掉跨级选项后重新归一化权重 function pickNextState(currentState, cfg) { const ADJACENT = { NORMAL: ["NORMAL", "INTERESTING"], INTERESTING: ["NORMAL", "INTERESTING", "REMARKABLE"], REMARKABLE: ["INTERESTING", "REMARKABLE"] }; const allowed = cfg.allowJump ? ["NORMAL", "INTERESTING", "REMARKABLE"] : ADJACENT[currentState]; return weightedRandom(cfg.stateWeights, allowed); }

七、与现有系统的衔接

替换 FOCUS_ACTIVE_WINDOW_DURATION_MS

现有代码用固定的 4000ms 超时驱动对焦窗口结束。焦内序列实现后，窗口结束改为"序列播放到 TRANSFER 节点"驱动，FOCUS_ACTIVE_WINDOW_DURATION_MS 降级为兜底最大值（防止异常情况下窗口永远不结束）。

现有机制焦内序列实现后
4000ms 计时器到期 → triggerFocusTimeout 序列到达 TRANSFER 节点 → triggerTransfer（与超时走相同的退出动画流程）
timeout → advancePhotoSequence → 可能飞走 transfer → 进入 REPOSITION（多数情况）或 LOST（少数情况，由外部序列状态决定）
固定 4000ms，不体现鸟类差异 序列总时长由生成结果决定，翠鸟短、苍鹭长，差异可感知

cardDraw.js 的变化

焦内序列实现后，拍摄时的 behaviorState 来自焦内序列的当前状态（badge 显示的状态），而非外部 photoSequence 的当前步骤。drawCard(speciesId, behaviorState) 的调用方式不变，传入的 behaviorState 参数来源更换。

cardDraw.js 中的 RARITY_WEIGHTS_BY_BEHAVIOR 混池逻辑可以同步移除，改为直接按 behaviorState 取对应 rarity 的卡（所见即所得）。

gameState 新增字段

// gameState 中新增，用于追踪焦内序列播放状态 currentFocusSequence: { segments: [], // 生成的序列 segmentIndex: 0, // 当前播放到第几段 safetyMs: 1200, // 安全区间 startedAt: 0 // 序列开始时间戳（用于安全区间计算） } | null
实现建议：焦内序列的状态播放逻辑放在 main.js 的 rAF 循环（updateFocusAnimation）中，每帧检查当前经过时间是否已超过当前 segment 的 durationMs，若超过则推进到下一 segment，并用新状态重新取 focusConfig 参数（现有的 focusKey 机制会自动触发 animation 重启）。序列播放到 TRANSFER 节点时调用 triggerTransfer，走与现有 timeout 相同的退出流程。
