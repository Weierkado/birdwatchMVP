1. 设计概览

现有的 section.status-grid 包含 6 个 .status-item 信息块，在探索和拍摄阶段始终保持相同布局。本次改动的核心是让这块区域在两种模式下呈现完全不同的内容——通过 CSS 3D 翻转实现从"探索信息面板"到"拍摄取景框"的切换。

正面 · 探索模式

5 个现有信息块（地点、时间/电量、天气、事件、方向）+ 第 6 块替换为小地图占位（内容待地图专题定）。

与现在最大的区别：「拍摄时机」块消失，换成地图。

背面 · 拍摄模式

整块区域变成大取景框：背景层 + 遮挡物层 + 鸟的剪影 + 行为 badge。四角取景框标记放大到整个区域边缘。

下方保留操控区（举起/快门/放下按钮）不参与翻转。

2. 顶部 UI 状态机

status-grid 状态 ├── EXPLORATION ← 默认状态，显示探索信息面板 │ 正面朝上，6 块信息，第 6 块 = 小地图占位 │ ├── FLIPPING_UP ← 举起相机触发（raiseCamera action） │ CSS rotateX(-180deg) 动画开始，约 0.4s │ ├── CAPTURE ← 翻转完成，显示大取景框 │ 背面朝上，全区域 = 取景框 + 剪影 + badge │ └── FLIPPING_DOWN ← 放下相机触发（putDownCamera action） CSS rotateX(0deg) 动画开始，约 0.3s，回到 EXPLORATION
触发点已有：raiseCamera → enterFocusPhase() 在 gameSession.js，putDownCamera → 退出对焦。只需在这两个时机加 class 切换，不需要改状态机逻辑。

4. 翻转动画

CSS 方案

使用 CSS 3D 变换，rotateX 轴翻转（向上展开像翻盖）。正面和背面各自设置 backface-visibility: hidden，互不穿透。

/* 容器：保持 3D 上下文 */ .status-grid { perspective: 800px; transform-style: preserve-3d; /* 高度在翻转后由背面内容撑开 */ } /* 内部翻转层，包裹正面和背面 */ .status-grid-flipper { position: relative; transform-style: preserve-3d; transition: transform 0.42s cubic-bezier(0.4, 0, 0.2, 1); } .status-grid-flipper.is-capture { transform: rotateX(180deg); } /* 正面（探索模式，6 块） */ .status-grid-front { backface-visibility: hidden; /* 保持现有 .status-grid 的 grid 布局 */ } /* 背面（拍摄模式，大取景框） */ .status-grid-back { position: absolute; inset: 0; backface-visibility: hidden; transform: rotateX(180deg); /* 预先翻转，翻转后正立 */ }

触发时机与代码接入点

// src/main.js — renderPhotoTimingStatus() 或 renderActions() 区域附近 // 在 enterFocusPhase() 调用之后挂钩 function onCameraRaised() { document.querySelector('.status-grid-flipper').classList.add('is-capture') } function onCameraLowered() { document.querySelector('.status-grid-flipper').classList.remove('is-capture') }

6. 鸟类剪影系统

设计决策

  • 剪影作为 <img class="bird-silhouette"> 与 .focus-moving-badge 并列放在同一父容器 .focus-bird-anchor 内——剪影在 badge 外部，不被 badge 边框或 overflow 裁切，可以向外延伸
  • 父容器 .focus-bird-anchor 统一执行 transform: translate(Xpx, Ypx)，badge 和剪影同步运动，无需额外动画代码
  • 左右方向通过 CSS scaleX(-1) 实现，不需要两套素材
  • 无序列帧，切换时直接替换 src 属性

剪影变体（per bird）

文件名后缀用途切换时机
_perch停栖姿态（默认）入场后即显示
_fly展翅飞走姿态photoPhase 进入 LOST / EXITING 时
_pose精彩动作姿态（可选）焦内序列进入 REMARKABLE 状态时
_pose 变体在功能框架阶段可以复用 _perch，等美术资产到位再替换，不影响功能验证。

左右方向规则

// updateFocusAnimation() 内，每帧根据 anchor 速度方向更新 const silhouette = anchor.querySelector('.bird-silhouette') if (vel.x > 0.5) { silhouette.style.transform = '' // 向右，默认朝向 } else if (vel.x < -0.5) { silhouette.style.transform = 'scaleX(-1)' // 向左，水平翻转 } // vel.x 在 ±0.5 之内时保持上一帧方向，避免快速抖动

飞走时的切换

// 在 enterExitingPhase() 或 LOST 状态触发时调用 function setSilhouetteFly(badge) { const img = badge.querySelector('.bird-silhouette') const currentSrc = img.src img.src = currentSrc.replace(/_perch|_pose/, '_fly') }

命名规范

assets/silhouettes/ kingfisher_perch.svg // 翠鸟 · 停栖 kingfisher_fly.svg // 翠鸟 · 飞走 kingfisher_pose.svg // 翠鸟 · 精彩（可延后） sparrow_perch.svg sparrow_fly.svg sparrow_pose.svg bluejay_perch.svg // 红嘴蓝鹊 bluejay_fly.svg bluejay_pose.svg mandarin_perch.svg // 鸳鸯 ...（6 鸟 × 2-3 变体 = 12-18 个文件）
剪影用 SVG 而不是位图——缩放无损，文件体积极小（每个约 1-3KB），可直接用 CSS fill 设置颜色，也容易做半透明叠加效果。
