1. 设计目标

本次迭代针对 MVP 二测暴露的两个核心问题：

问题一：妹妹回复链路过深

拍到照片后，看到妹妹回复需要：打开笔记 → 找到对应鸟 → 找到照片 → 发给妹妹 → 等回复 → 回来看。情绪峰值在拍完瞬间，路径太深时冲动已经消失。

解法：RESULT 阶段直接在行动面板加「发给妹妹」按钮，妹妹角标有新回复时一跳直达。

问题二：局外成长感缺失

笔记页当前是卡牌陈列，没有时间感和手写质感；局与局之间缺少沉淀节点，拍过的鸟和拍过的感受等同于数据条目。

解法：熟悉度系统 + 夜晚整理模式，把当日拍摄转化为笔记页上逐渐丰满的手账文字。

所有改动在现有 MVP 代码上增量叠加。不重建任何已运转的系统，不新增独立场景文件——全部功能均通过 state.mode 切换 + main.js 新增渲染函数实现。

2. 新局外循环

EXPLORE → 拍鸟 → PHOTO · RESULT 阶段 ↓ （可选）行动面板「发给妹妹看看」按钮 → 调用现有 sendCollectedCardToSister() ↓ 当日照片推动各鸟熟悉度进展（由 familiaritySystem.js 在卡牌入图鉴时计算） ↓ 电量耗尽 → 进入 SETTLEMENT mode（现有结算界面） ↓ 结算页新增入口：「整理今天的观察 →」（有待补充槽时高亮） ↓ 进入 NIGHT_REVIEW mode（新增） ↓ 待补充笔记槽列表 → 点选 → 全文预览 → 确认写入 ↓ noteEntry 写入对应 CollectedCardEntry，fieldGuide 存档持久化 ↓ FIELD_GUIDE mode · 笔记 Tab：文字累积，比昨天更丰满 ↓ 某只鸟熟悉度满格 → 「你觉得你已经认识它了。」里程碑 + 发给妹妹入口
局外成长的感知来源有两个：① 夜晚整理时「今天又写了一点」的仪式感；② 打开笔记页看到文字比昨天多。两者叠加才能建立「我的笔记在生长」的感受，缺任何一个都会打折。

5. 熟悉度系统

每只鸟独立一条熟悉度进度（存于 fieldGuide.familiarityBySpecies），是局外可感知的成长指标，同时决定每次可补充的夜晚笔记数。

推进规则（由 familiaritySystem.js 在 addCollectedCard 时触发）

情况 A · 新卡槽首次解锁

该鸟的某张 card 首次被收入 collectedCards，无论对焦质量。每只鸟最多触发 6 次（3 NORMAL + 2 INTERESTING + 1 REMARKABLE）。

情况 B · 对焦档位刷新最佳记录

某张已有的 snapshot 被更高 focusScore 的新照片刷新最佳记录（失焦→尚可→清晰→数毛）。同一卡槽最多触发 3 次。

熟悉度与夜晚笔记槽的关系

每次推进时，在 gameState.pendingNoteSlots（session 级，不持久化）里追加一条待补充槽。当日有效推进次数 = 当晚可补充的笔记条数上限，设计目标区间：2–4 条 / 天。

满格条件

  • 该鸟 6 个卡槽全部解锁，且
  • 至少 4 个卡槽的最佳 snapshot focusScore ≥「清晰」档

满格时 familiarity.isComplete = true，笔记页出现里程碑文案和「发给妹妹」入口。

6. 夜晚整理模式（NIGHT_REVIEW）

新增 state.mode

'NIGHT_REVIEW' 加入现有 mode 列表。进入条件：从 SETTLEMENT 页点击「整理今天的观察」，或从顶部夜晚角标点击（有待补充槽时）。

// src/gameSession.js — 新增函数 export function startNightReview(state) { state.mode = 'NIGHT_REVIEW' // pendingNoteSlots 在局内已由 familiaritySystem 填充，此处不需要重建 // 若 pendingNoteSlots 为空（游戏重启后），从 fieldGuide 重建： // 扫描 collectedCards 中 noteEntry === null 的已解锁卡，重建 pendingNoteSlots }

NIGHT_REVIEW 界面流程

进入 NIGHT_REVIEW → renderNightReview() 渲染到 elements.detailPanel ↓ 显示当日待补充笔记槽列表 （每条：鸟种名 · 行为稀有度 · 对焦档位 · 卡牌 description 前 14 字） ↓ 玩家点击某条 → 展开全文预览 （显示完整 card.description，背景弱化其余槽） ↓ 确认「写入笔记」 ↓ CollectedCardEntry.noteEntry 写入（text/rarity/writtenAt/layoutOffset） fieldGuide 存档持久化（saveFieldGuide） ↓ 写入动画：文字逐字显现（打字机效果，约 0.8s） ↓ 可继续补充其余槽，或点击「回到今天」退出 NIGHT_REVIEW → 回到 SETTLEMENT ↓ 全部补完时收尾文案：「今天又多认识了一点。」
「未补充的槽保留」——玩家忙时不会丢失进度，但积累越多，整理负担越大，自然形成「尽量每天写一点」的节奏压力。重启游戏后从 collectedCards 重建 pendingNoteSlots，不依赖 session 内存。

7. 笔记 Tab 重构

布局原则

从「照片卡片展示页」改为「手账式观察记录页」。照片全部移到相册 Tab，笔记 Tab 只有文字和熟悉度。

单只鸟的笔记区结构

┌─────────────────────────────┐ │ 麻雀 ●●●○○ │ ← 鸟名 + 熟悉度进度点（5 格 = score/maxScore 比例） │ │ │ 它低头啄了两下， │ ← 已写入的 noteEntry.text │ 动作很快，总让你 │ │ 以为它还没吃饱。 │ │ │ │ （留白） │ ← 未写入的槽，隐形占位 │ │ │ [涂鸦] │ ← P2：熟悉度达阈值后解锁 └─────────────────────────────┘

文字排布规则

  • 每条 noteEntry 自然折行，不同行数产生高低起伏节奏感
  • 文字位置有 noteEntry.layoutOffset（±4px，补充时 roll 并持久化）的轻微随机偏移，模拟手写排版
  • REMARKABLE 来源的文字：字号 +1px 或行距略宽（CSS class 区分）
  • 文字颜色使用对应鸟种的主色调（浅色调，不影响阅读）

满格里程碑

当 familiarity.isComplete === true 时，笔记页该鸟区域底部出现：

「你觉得你已经认识它了。也许可以告诉妹妹。」

文字下方出现「发给妹妹」按钮（调用现有发送链路，但触发专属回复剧情；completeSentToSister 标记防重复）。

手绘涂鸦（P2）

触发条件涂鸦内容位置
该鸟首次加新（score > 0）鸟的简笔轮廓（极简线稿）区域右上角
score ≥ maxScore × 50%特征细节（嘴形、羽色标注线）轮廓旁
isComplete完整涂鸦（带色块，手绘质感）覆盖前两阶段

doodleStage（0/1/2/3）由 familiarity.score 派生计算，不单独存储。
