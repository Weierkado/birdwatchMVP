原型设计说明

核心决策：先把底部栏、消息 overlay、笔记 overlay 的交互跑顺，再讨论夜晚整理和发布反馈如何接回完整流程。

这轮对齐了什么

A 主界面去掉“今天的目标”和主界面的“查看消息 / 打开笔记”大按钮，回到当前游戏更接近的顶部状态区、事件描述和行动按钮结构。
B 工具层消息和笔记继续使用底部抽屉，但不会盖住底部导航。底部导航只保留观察 / 消息 / 笔记三个入口。
C 二级页消息二级页改为在同一张消息 sheet 内横向滑入；笔记卡牌详情改为在笔记 sheet 内浮出较轻的悬浮弹窗，不再把用户带离当前工具上下文。

当前交互判断

  • 底部栏：只承担观察、消息、笔记三种快速切换，不承载夜晚整理。
  • 消息二级页：从消息列表进入聊天时，在同一张 overlay 内横向切页，返回时也留在这张工具层里。
  • 笔记二级页：卡牌详情改为笔记 overlay 内的悬浮弹窗，保留原列表、纸感背景和当前上下文。
  • 夜晚整理：保留在演示里，但只通过顶部调试按钮进入。

这轮仍然没做的事

  • 不接入真实数据。
  • 不改正式游戏代码。
  • 不把消息系统做成完整社交应用。
  • 不提前设计完整三测笔记体系。
