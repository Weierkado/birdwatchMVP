陀螺仪对焦微调

对焦阶段 · 倾斜手机轻微移动绿框 · 解决「鸟在框边一点点」的束手无策感

PHOTO · FOCUS phase MVP 微交互 DeviceOrientationEvent 2026-06-11

1. 功能定位

对焦阶段，badge 在一条固定轨迹上移动，绿框目前完全静止。极端情况下 badge 轨迹恰好压着绿框边缘，玩家只能眼睁睁看着「差一点」——本功能用轻微的物理倾斜来给玩家一个最后的微调手段。

定位是「兜底微调」，不是「主动追踪」。绿框的最大位移很小，玩家无法靠倾斜完全追着 badge 走；badge 的大半段轨迹仍由时机判断决定。物理感来自「瞄准相机」的隐喻，端稳手机才能对准。

不需要做的事：不新增任何 UI 提示，不显示「陀螺仪已启用」字样；无陀螺仪或权限拒绝时绿框保持固定，静默降级，体验与当前一致。

2. 关键参数

GYRO_MAX_OFFSET
22 px
绿框最大偏移量（单轴）。占位值，测试后调整。
GYRO_DEAD_ZONE
2 °
死区角度，小于此值的倾斜不产生移动，过滤握持抖动。
GYRO_SENSITIVITY
1.3 px/°
死区外每度对应的像素位移。±22px ÷ 1.3 ≈ 需要倾斜约 17° 到达极限。
GYRO_SMOOTH_ALPHA
0.2
指数平滑系数（EMA）。越小越平滑但响应越慢；0.2 是「跟随感」的经验起点。
为什么 GYRO_MAX_OFFSET = 22px：CAMERA_FOCUS_GREEN = 60，绿框宽高各 60px。22px ≈ 绿框边长的 37%，即玩家最多能把绿框拖出「中心 + 半个框身」的范围，足以救回压边的 badge，但不至于让玩家靠倾斜追整段轨迹。

3. 技术实现

API 选择

使用 DeviceOrientationEvent（不是 DeviceMotionEvent）。它给出角度值，比加速度更稳定，适合持续的位置偏移控制。关键字段：

  • event.gamma — 手机左右倾斜，-90° ~ 90° → 映射到水平偏移 X
  • event.beta — 手机前后倾斜，-180° ~ 180° → 映射到垂直偏移 Y

iOS 权限（必须处理）

iOS 13+ 的 DeviceOrientationEvent.requestPermission() 必须在用户手势回调里调用，否则静默失败。时机：玩家进入 FOCUS phase 时（已有按钮触发，是合法的用户手势上下文）。权限结果存 sessionStorage，同局不重复请求。

不要在页面 load 时或非手势场景调用 requestPermission，iOS 会直接拒绝且不弹窗。

校准

每次进入 FOCUS phase 时记录当前 beta / gamma 作为基准（baseline）。所有偏移量相对 baseline 计算，与玩家握持姿势无关。REPOSITION 后再次进入 FOCUS 时重新校准。

核心逻辑（src/gyroInput.js）

// 单例，由 main.js 初始化 const GyroInput = { _handler: null, _baseline: null, _smoothX: 0, _smoothY: 0, onOffset: null, // callback(offsetX, offsetY) async enable(onOffset) { // iOS 权限门 if (typeof DeviceOrientationEvent?.requestPermission === 'function') { const cached = sessionStorage.getItem('gyroPermission') if (cached !== 'granted') { const res = await DeviceOrientationEvent.requestPermission() sessionStorage.setItem('gyroPermission', res) if (res !== 'granted') return // 静默降级 } } if (!('DeviceOrientationEvent' in window)) return // 桌面端静默降级 this.onOffset = onOffset this._baseline = null this._smoothX = 0 this._smoothY = 0 this._handler = this._handle.bind(this) window.addEventListener('deviceorientation', this._handler) }, disable() { if (this._handler) window.removeEventListener('deviceorientation', this._handler) this._handler = null this._baseline = null }, _handle(e) { if (!this._baseline) { this._baseline = { beta: e.beta, gamma: e.gamma } // 首帧校准 return } const dg = e.gamma - this._baseline.gamma // 左右倾斜差值 const db = e.beta - this._baseline.beta // 前后倾斜差值 const rawX = Math.abs(dg) > GYRO_DEAD_ZONE ? dg * GYRO_SENSITIVITY : 0 const rawY = Math.abs(db) > GYRO_DEAD_ZONE ? db * GYRO_SENSITIVITY : 0 // EMA 平滑 this._smoothX += GYRO_SMOOTH_ALPHA * (rawX - this._smoothX) this._smoothY += GYRO_SMOOTH_ALPHA * (rawY - this._smoothY) // 限幅 const clamp = (v, max) => Math.max(-max, Math.min(max, v)) this.onOffset?.( clamp(this._smoothX, GYRO_MAX_OFFSET), clamp(this._smoothY, GYRO_MAX_OFFSET) ) } } export default GyroInput

focusEngine.js 改动

对焦质量判定从固定中心坐标改为接受当前绿框偏移量：

// 修改前：绿框中心固定，badge 位置直接与常量比较 // 修改后：判定时加入 gyroOffset function isBadgeInFocusZone(badgePos, gyroOffset) { const zoneCenterX = FOCUS_CENTER_X + gyroOffset.x const zoneCenterY = FOCUS_CENTER_Y + gyroOffset.y const half = CAMERA_FOCUS_GREEN / 2 return Math.abs(badgePos.x - zoneCenterX) < half && Math.abs(badgePos.y - zoneCenterY) < half }

绿框渲染

绿框 DOM 元素（.focus-zone 或对应元素）在 onOffset 回调里更新 CSS transform：

gyroInput.enable((ox, oy) => { state.gyroOffset = { x: ox, y: oy } focusZoneEl.style.transform = `translate(${ox}px, ${oy}px)` })
依赖 TOP_UI_CAMERA_REDESIGN · TU-D01（DOM 迁移）：.focus-playfield 和绿框元素将从 #photoTimingText（status-grid 正面）迁移到 .viewfinder-bird-layer（status-grid 背面翻转容器）。GY-C03 实现时 focusZoneEl 的查询选择器必须使用迁移后路径（.viewfinder-bird-layer .focus-zone），不能继续用 #photoTimingText .focus-zone。建议 GY-C03 安排在 TU-D01 完成后实施。

Y 轴方向需实机确认：.viewfinder-bird-layer 位于 rotateX(180deg) 的 3D 翻转容器内，gamma → X 偏移方向应正常，但 beta → Y 的屏幕视觉上下对应关系可能与直觉相反（3D 翻转后元素本地 Y 轴与屏幕 Y 轴反向）。联调时必须在真机举起相机状态下测试「向前倾手机，绿框向上走」是否符合预期，必要时给 oy 加负号。

4. 生命周期

时机动作
进入 photoPhase === 'FOCUS'调用 gyroInput.enable(onOffset)；重置 gyroOffset 为 (0, 0)；首帧自动校准
FOCUS → REPOSITION（badge 转移）gyroInput.disable()；绿框 transform 清零
REPOSITION → 再次 FOCUS重新调用 enable()，重新校准
FOCUS → RESULT（快门按下）gyroInput.disable()；isBadgeInFocusZone 用当前 gyroOffset 判定最终对焦质量
退出 PHOTO phase（LOST / 结算）gyroInput.disable()
每次重新进入 FOCUS 都重新校准，避免玩家在 REPOSITION 期间调整了握姿导致下一次 FOCUS 绿框突然偏移。

5. 降级行为

情况行为
桌面浏览器（无陀螺仪）enable() 内检测失败，直接 return；绿框固定不动
iOS 用户拒绝权限同上，静默降级；不弹二次提示
Android（无需权限）正常工作
手机平放（beta ≈ 0）死区吸收，绿框不动；轻微倾起后才开始响应
