6. 开发注意事项

data/config.js 批量追加（Day 1 一次性完成）

四个系统共计新增约 15 个常量，建议 Day 1 一次性全部追加，按系统分组注释，避免后续多次触碰同一文件产生冲突：

  • GYRO：4 个（GYRO_MAX_OFFSET / GYRO_DEAD_ZONE / GYRO_SENSITIVITY / GYRO_SMOOTH_ALPHA）
  • EVENT：4 个（EVENT_HINT_COOLDOWN_TURNS / DISPLAY_MS / FLASH_MS / QUEUE_MAX）
  • WEATHER：3 个常量 + WEATHER_WEIGHTS 权重对象

gameState.js 同步追加（Day 1 一次性完成）

createDefaultGameState() 新增 3 个字段（gyroOffset / weather / pendingNoteSlots），建议同 config.js 在 Day 1 一次性追加。

Y 轴方向联调（GYRO）

.viewfinder-bird-layer 位于 rotateX(180deg) 3D 容器内，beta → Y 的屏幕视觉方向可能与直觉相反。TU-D01 + GY-C03 完成后，必须在真机举起相机状态下验证「向前倾手机，绿框向上走」，必要时给 oy 加负号。

#spotText 类型变更（WEATHER WE-U01）

将 #spotText 从 button 改为 span。确认 renderStatusBlocks() 中已有天气文字初始渲染逻辑（不需要 WE-C01 再重写）；weatherSystem.js 只需在天气变化时做直接 DOM 更新，不重写整块状态渲染。

NIGHT mode 第三分支（NOTES NR-C03）

实现 NIGHT_REVIEW 时，需在 render() 的 moment 槽逻辑中加第三分支（else if (state.mode === 'NIGHT_REVIEW')），与 TU-MAP01 预留的双态结构对接。

存档字段向下兼容

normalizeCollectedCards() 已有空值补全机制，noteEntry: null 追加后旧存档自动兼容。familiarityBySpecies: {} 在 createDefaultFieldGuide() 中作为缺省值，旧存档缺失时由 storage.js 补全即可。
