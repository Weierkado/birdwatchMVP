对焦词缀系统 · 设计文档 v2

三档简化为两档（失焦 / 正常），保留图鉴解锁，调整积分权重
测试版本：下一版 词缀档数：2（失焦 / 正常） 机制变动：移除毕业档 鸟种参数：文案专题落地

一、迭代概述

上一版设计为三档词缀：丢焦 / 尚可 / 毕业。本次迭代合并为两档：

v1 词缀 v2 词缀 判定来源 变更
丢焦（badge 在绿框外） 失焦 badge 在绿框外点击 重命名，规则不变
尚可（badge 在绿框内） 正常 badge 在绿框内点击（任意位置） 合并为同一档
毕业（badge 在精准区内）
移除毕业档是为了降低首次测试对焦玩法的认知负担。玩家只需要判断「有没有对上焦」，不需要同时追求「毕业精准区」。精准档可在后续版本作为可选深度目标重新引入。

// v2 简化判定
function getFocusAffixFromPosition(distance, config) {
  return distance <= config.focus.green ? 'NORMAL' : 'BLUR'
}

三、积分规则

3.1 基础积分（行为状态）

行为状态（badge 稀有度） 正常积分 失焦积分（-1） 图鉴解锁
寻常（NORMAL） 1 0 ✓ 解锁
有趣（INTERESTING） 2 1 ✓ 解锁
精彩（REMARKABLE） 3 2 ✓ 解锁

积分下限为 0，不出现负分。失焦照片仍然计入图鉴词条解锁，但不计入收集完成度（见下）。

边界情况：寻常×失焦 = 0 分。玩家消耗了一格电量、拍了一张照片，积分收益为零。图鉴词条如果已解锁，则这张照片对当局完全无价值。

这不一定是问题——失败有代价是合理的——但测试时留意：玩家是否会反复在寻常状态下失焦、并对"拍了等于没拍"感到困惑或挫败，而非动力重新尝试。

3.2 收集规则

行为结果
任意状态 × 任意词缀（含失焦）首次拍到某鸟种图鉴词条解锁
失焦照片收录进本局照片列表，计入拍摄张数，扣除电量，积分按上表
正常照片同上，加完整积分
解锁与积分分离的好处：玩家第一次遭遇某鸟，哪怕失焦也能留下「我见过这只鸟」的记录，不会因为操作失误丢失首次遭遇的文本体验。积分的差异则持续引导玩家提升对焦技巧。

四、失焦的视觉差异化

4.1 设计原则

失焦标记需要满足：

  • 玩家在结算和收藏界面能一眼区分失焦与正常照片
  • 不喧宾夺主——失焦照片仍有信息价值（鸟种文本），不应被遮蔽
  • 与文字游戏的排版风格一致，避免引入图形元素

「失焦」作为卡片属性 badge，与稀有度（有趣 / 寻常 / 精彩）并列显示在卡牌头部信息行。鸟种名称本身不携带任何标注，保持干净。

方案实现优点缺点
A（确认） 稀有度 badge 后追加「失焦」badge，同行显示（<span class="affix-blur-tag">失焦</span>） 语义清晰：失焦是卡牌属性而非鸟种属性；排版无需额外层级；与稀有度 badge 视觉系统统一 badge 排列过多时（如未来增加更多词缀）需要换行处理
备选 右上角文字角标（absolute 定位） 在列表浏览时扫视右上角可快速筛选 需要 position:relative 容器；与主信息区分离，视觉联系弱
方案 A 已确认。失焦是对这张照片的判定，与「有趣」是对这次时机的判定，两者平级。放在同一信息行符合层级逻辑，实现也最简洁——正常照片不渲染该 badge，失焦照片追加一个 span，不改变其他布局。

5.3 参数设计指引（供文案专题参考）

鸟类性格维度对应参数参考方向
活跃 / 急促（如翠鸟）pattern: jitter / sweep，高振幅高频率绿框可适当收窄（55–65px），badge 移动快，窗口窄
沉静 / 缓慢（如苍鹭类）pattern: drift_to_center / wander，低振幅低频率绿框可适当放宽（65–80px），badge 移动慢，窗口宽
偶尔抖动（如麻雀）pattern: wander + 偶发 stutter中等绿框（60–70px），节奏感来自不规律性而非速度
REMARKABLE 状态通常比 NORMAL 有更高振幅或更快频率精彩时机对焦更难，符合「珍贵 = 有代价」直觉

六、设计风险与后续

失去「毕业」后的成就感缺口。三档系统中「毕业」是明确的峰值体验——玩家知道自己做到了最好。两档系统移除了这个时刻，正常就是终点，缺少「超额完成」的满足感。

短期测试阶段影响有限，但若玩家反馈「对焦感觉差不多，拍上就行」，则可能需要重新引入毕业档，或通过动画强化「正常判定瞬间」的反馈来弥补峰值体验。
