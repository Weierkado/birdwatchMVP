观鸟模拟器 · 全局框架文档

定位：HTML竖屏手机游戏 · 项目开发能力验证 · 概念筹码
平台 HTML / 移动端竖屏
开发方式 Vibecoding
预计游玩时长 装备拉满 1-1.5h · 全收集 3h
目标受众 圈子内测 / 投资方演示 / 美术招募

项目定位

在 3 分钟局内，通过翻牌找鸟、抓住合适时机按下快门，收集尽可能多鸟种与高稀有度卡牌。

核心验证目标：大循环框架的决策合理性（找鸟选择 · 拍照时机 · 卡槽管理）。微观体验（真实观鸟沉浸感）将在后续 Godot 原型中验证，本项目不承担此目标。

功能优先级

优先级 模块 理由 状态
P0 局内找鸟玩法 核心体验，决策验证的主体 必须完成
P0 图鉴 跨局积累感，决定是否有理由继续玩 必须完成
P1 地图选点 + 预听 增加前置决策层，需 P0 稳定后叠加 P0 完成后评估
P2 局外养成（星星 / 装备） 体验优化，非验证必要 时间允许则做
P2 结算演出 逐张揭示增强情感，实现简单 优先于装备系统
创意工坊类功能超出 HTML demo 范围，不纳入本次开发。

模块划分与接口

GameSession
局内总控。管理计时器、SD卡槽状态、局内事件派发。
对外事件 session:start session:end photo:taken bird:call 对外接口（预留） init(spotConfig) — 接收地图传入的驻点配置（鸟种池、密度参数）
CardGrid
4面墙 × 4×4格子。管理卡片状态、翻牌动画、墙面切换。
对外接口 markCardActive(wall, idx) clearCard(wall, idx) flipCard(wall, idx) 对外事件 card:flipped card:flippedEmpty
BirdManager
全局鸟类实体管理。控制入场、移动（跨墙）、离场，驱动卡片动效。
对外事件 bird:moved bird:calling bird:left 数值参数 同时在场 3-4 只 · 平均停留 ~30s
BehaviorSequence
鸟类拍照模式下的行为序列生成与播放。种子驱动，管理各时段状态。
对外接口 start(seed) getCurrentState() onFlyAway(cb) 时段枚举 NORMAL INTERESTING REMARKABLE FLY_AWAY
PhotoMode
拍照子流程。放大卡片、展示行为色、处理快门逻辑、结束回归网格。
对外事件 photo:captured(behaviorState) photoMode:ended(reason) 结束原因枚举 SLOTS_FULL BIRD_LEFT CANCELLED
SDCard
局内存储管理。记录本局所有照片，局结束后供结算读取。
对外接口 addPhoto(photoData) isFull() getAll() 数值 容量 50 张 · 每局重置
AudioSystem
播放真实鸟叫音频。不做空间音效，方位由视觉系统处理。
对外接口 playCall(speciesId) 触发方 BirdManager 在鸟叫时调用
DirectionHUD
其他墙鸟叫方位视觉化。根据鸟的绝对墙位与玩家当前朝向实时计算相对方位，在对应屏幕边缘显示声波特效。
对外接口 showCallEffect(birdAbsWall, facingWall) 方位计算 relDir = (birdAbsWall - facingWall + 4) % 4
relDir 1→右边缘 · relDir 2→下边缘 · relDir 3→左边缘
CardDraw
根据当前行为时段从对应卡池抽取卡牌。
对外接口 draw(speciesId, behaviorState) → CardData 卡池规则 NORMAL→常态池 · INTERESTING→有趣+常态池 · REMARKABLE→全池
FieldGuide
图鉴数据持久化与展示。记录听到、拍到的鸟种状态。
对外接口 markHeard(speciesId) addCard(cardData) render() 状态枚举 UNKNOWN（?） HEARD（剪影） DISCOVERED（完整）
MapScene [P1]
驻点选择地图。预留接口供 P1 阶段接入。
预留输出接口 SpotConfig { speciesPool, density, audioPreview } → GameSession.init()
MetaProgression [P2]
星星货币与装备。预留 SD 卡槽数量覆盖接口。
预留接口 getSlotCount() → number → SDCard 初始化时读取（当前 hardcode 50）

核心数据结构

Bird（鸟类实体）

// 运行时鸟类实体，全局唯一，按 id 管理 { id: string, // 本次出场唯一 id speciesId: string, // 鸟种 id，关联 SpeciesData wall: 0|1|2|3, // 当前所在墙面（0前1右2后3左） cardIndex: number, // 当前所在格子索引 (0-15) entryTime: number, // 入场时间戳 plannedStay: number, // 预计停留时长（ms），平均 30000 sequenceSeed: number // 拍照行为序列种子，入场时预生成 }

CardState（卡片状态）

{ wall: number, index: number, state: 'idle' | 'active' | 'flipping' | 'photo', // idle: 空卡正面朝下 // active: 有鸟在此，背面有动效 // flipping: 翻牌动画中 // photo: 已放大进入拍照模式 birdId: string | null // 当前占用的鸟类实体 id }

PhotoData（单张照片）

{ id: string, speciesId: string, behaviorState: 'NORMAL' | 'INTERESTING' | 'REMARKABLE', cardId: string, // 抽到的卡牌 id capturedAt: number // 本局内时间戳（用于结算排序） }

CardData（卡牌定义）

{ id: string, speciesId: string, behaviorTier: 'NORMAL' | 'INTERESTING' | 'REMARKABLE', rarity: 1|2|3|4|5, // 星级，决定颜色与图鉴价值 behaviorName: string, // 行为名，如"梳理羽毛" specialName: string | null, // 部分卡有特殊名称 artKey: string // 美术资产索引 }

SpeciesData（鸟种定义）

{ id: string, name: string, audioFile: string, // 鸟叫音频文件路径 baseDensity: number, // 在局内的出现概率权重 cardPool: { // 各时段可出的卡牌 id 列表 NORMAL: string[], INTERESTING: string[], REMARKABLE: string[] } }

关键预留接口

MAP → SESSION（P1 地图选点接入点）
MapScene 选点完成后调用 GameSession.init(SpotConfig)，传入该驻点的鸟种权重池与密度参数。
P0 阶段 GameSession 使用内置默认配置，此接口仅声明，不调用。
SESSION → FIELD GUIDE（实时图鉴更新）
AudioSystem 播放鸟叫时同步触发 FieldGuide.markHeard(speciesId)。
SDCard.addPhoto() 时同步触发 FieldGuide.addCard(cardData)。
两者均为单向推送，FieldGuide 被动接收，不需要轮询。
META → SDCARD（P2 装备接入点）
SDCard 初始化时调用 MetaProgression.getSlotCount() 获取当前卡槽上限。
P0/P1 阶段此函数 hardcode 返回 50，P2 接入后替换为实际养成值。
SESSION → SETTLEMENT（结算数据移交）
局结束后，GameSession 将 SDCard.getAll() 返回的照片列表传给结算界面。
结算按 capturedAt 排序后逐张揭示，揭示完成后跳转图鉴。
