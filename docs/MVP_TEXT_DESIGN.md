# 观鸟模拟器 · 纯文字 MVP 设计文档

## 1. 项目定位

本项目是一个纯 HTML + CSS + JavaScript 开发的观鸟模拟器文字验证版。

当前目标不是制作完整美术版游戏，也不是实现复杂动画、声音、4×4 网格或地图系统，而是验证核心玩法循环是否成立：

玩家根据文字线索寻找鸟类，在有限时间内判断是否观察、等待或拍照，通过拍照时机获得不同稀有度的鸟类卡牌，并在局后更新图鉴。

## 2. MVP 核心目标

本 MVP 需要验证以下问题：

1. 玩家是否能理解「线索 → 找鸟 → 发现鸟 → 拍照」的流程。
2. 玩家是否会在「立刻拍照」和「等待更好时机」之间产生决策。
3. 鸟类飞走、错过机会是否能产生轻微紧张感。
4. 卡牌稀有度和行为差异是否能驱动重复游玩。
5. 图鉴解锁是否能提供跨局收集动力。

## 3. 当前阶段不做的内容

以下内容属于后续阶段，不在文字 MVP 中实现：

- 4 面墙 × 4×4 卡片网格
- 翻牌动画
- 滑动切换墙面
- 真实鸟叫音频
- 声波方向 HUD
- 地图选点
- 预听系统
- 装备系统
- 星星商店
- 创意工坊
- 复杂结算演出
- 大量鸟种和大量卡牌
- React / Vue / Vite / TypeScript

本阶段只实现原生 HTML、CSS、JavaScript 版本。

## 5. 一局游戏的基本形式

原始设计中一局为 3 分钟倒计时。  
文字 MVP 暂时改为回合制：

- 1 局 = 30 回合
- 1 回合 ≈ 6 秒
- 30 回合 ≈ 180 秒，即 3 分钟

这样可以避免一开始处理实时倒计时、动画同步和复杂异步逻辑。

## 6. MVP 游戏循环

完整循环如下：

```text
开始游戏
↓
初始化局内状态
↓
生成若干只活动鸟
↓
玩家看到当前方向与文字线索
↓
玩家选择行动
↓
可能发现鸟 / 错过鸟 / 获得更强线索
↓
发现鸟后进入拍照序列
↓
玩家选择拍照、等待或放弃
↓
根据当前行为阶段抽取卡牌
↓
照片存入 SD 卡
↓
图鉴更新
↓
回合耗尽后结算
↓
展示本局拍到的照片与新增图鉴
