一、实现方式推荐

推荐：Google Looker Studio（原 Data Studio）
免费、与 Google Sheets 原生打通，无需写代码，图表类型覆盖全部需求。直接将 Sheets 作为数据源连接，每次打开看板自动刷新。

访问入口：lookerstudio.google.com → 新建报告 → 添加 Google Sheets 数据源 → 选择已有表格。

如果后续希望把看板嵌入网页或做更深度的自定义，可以改用 Chart.js + fetch Google Sheets 公开 CSV 的方案。但对于 20 人测试规模，Looker Studio 完全够用。

方案成本定制自由度适合阶段
Looker Studio免费，零代码中（模板驱动）当前测试阶段 ✓
Chart.js + Sheets CSV需编写 HTML高（完全自定义）8 月展会前后
Notion / Airtable 嵌入图表免费或低价低如果已在用 Notion
数据拆分建议：Sheets 里建议维护两张表。raw_events 存原始事件流（每行一个事件），survey_answers 存问卷答案（每行一个玩家）。看板连接这两张表，部分图表做 JOIN 时在 Looker Studio 里用"混合数据"功能实现，或在 Sheets 里预先用 VLOOKUP 打通 tester_id。

三、开发者本地测试数据过滤

本地调试和功能测试产生的数据会污染真实玩家数据，需要在看板层面排除。不需要在游戏里加额外入口，用 tester_id 前缀约定即可。

约定规则：开发者自己测试时，tester_id 统一使用 DEV_ 开头（如 DEV_YX、DEV_func、DEV_stress）。普通测试者的 ID 不会以 DEV_ 开头，自然隔离。
场景建议 tester_id说明
日常功能验证DEV_YX最常用，固定用这一个即可
数值压测DEV_stress如果需要区分压测记录
某次 bug 复现DEV_debug_1临时用，方便事后定位

在 Looker Studio 里配置排除过滤

在全局控制栏里再加一个固定过滤器（非用户可见的控件，直接写在数据配置里）：

  1. 打开数据源配置 → 添加过滤条件
  2. 条件：tester_id 不包含 DEV_
  3. 将此过滤器应用于所有图表

这个过滤是后台静默生效的，不显示在用户可见的控件里。开发者看完整数据时，可以临时把这条过滤删掉或改为"包含 DEV_"来专门审查自己的测试记录。

注意：如果某次你忘记填 tester_id（直接跳过测试入口进入游戏），那局数据不会带 tester_id 字段，在 Sheets 里这一行的 tester_id 为空。可以在 Looker Studio 里额外加一条过滤：tester_id 不为空，避免无 ID 的孤立数据干扰分析。或者在 Sheets 里手动补一行标注 DEV_manual。

全局筛选器

在 Looker Studio 报告顶部加一个 testerLevel 筛选下拉菜单，选中某个身份级别后，所有看板上的图表同步过滤到该群体。这样可以快速查看"如果只看资深观鸟者，整体反馈如何"。
