# 二测后项目整理审计（2026-06）

本审计为只读整理，用于后续清扫周拆分和回归计划。结论基于当前工作区代码与 `docs/DevLog/CODE_MAP.md`；未运行浏览器手测，运行时行为中未能从静态代码直接确认的部分标记为“待确认”。

## 一、项目入口

### index.html 直接入口

- `index.html` 只直接执行一个脚本：`<script type="module" src="./src/main.js"></script>`。
- 其余模块通过 `main.js` 的 ES module import 图加载，不是独立 `<script>` 执行。
- `<head>` 中有 `modulepreload`：`src/main.js`、`src/gameSession.js`、`src/gameState.js`、`src/birdManager.js`、`src/encounterSystem.js`、`src/photoSequence.js`、`src/focusEngine.js`、`src/focusSequence.js`、`src/fieldGuide.js`、`src/cardDraw.js`、`src/rarityDisplay.js`、`src/spotManager.js`、`src/storage.js`、`data/cards.js`、`data/species.js`、`data/config.js`、`data/spots.js`、`data/focusConfig.js`。

### data/config.js 加载顺序

- `data/config.js` 没有作为单独脚本标签执行，但被 `src/main.js`、`src/analytics.js`、`src/gameState.js`、`src/gameSession.js`、`src/birdManager.js`、`src/encounterSystem.js`、`src/photoSequence.js`、`src/focusEngine.js` 等模块直接或间接 import。
- ES module 语义保证依赖模块先于依赖方执行，因此 `config.js` 对 `main.js` / `analytics.js` 的当前使用没有明显顺序风险。
- `modulepreload` 只影响预取，不决定执行顺序。当前 `data/config.js` 的 preload 排在部分 data 文件之后，但这不影响 import 执行正确性。

### 当前风险

- `index.html` 的 `modulepreload` 列表需要手工维护；新增模块若忘记 preload，不会破坏功能，但可能影响首屏加载性能。
- `message-editor.html` 独立 import `src/liyaMessageSystem.js`，不经过 `index.html` 的 preload；这是开发工具入口，不影响游戏主入口。
- 待确认：静态服务路径必须能让 `fetch("data/liyaMessages.json")` 正确解析；直接以 `file://` 打开时可能有浏览器限制。

## 二、main.js 职责地图

`src/main.js` 当前约 6710 行，是页面级编排中心。它不是纯渲染层，实际承担了 UI、运行时动画、消息、图鉴、结算、survey 和 analytics 协调。

### 初始化

- 创建 `gameState = createDefaultGameState()`。
- 查询固定 DOM 到 `elements`。
- 运行时创建 `.utility-actions` 与 `.reset-actions`。
- 读取观鸟日序号 `birdwatch_text_sim_day_index`。
- 启动 `loadLiyaMessages().then(render)` 的消息文本预加载刷新。

### 数据加载

- 静态 import：`cardList`、`speciesList`、`SISTER_KNOWLEDGE_BY_CARD`、`SISTER_KNOWLEDGE_FALLBACK`、`INITIAL_MESSAGE_THREADS`、配置常量。
- 动态 fetch 间接由 `loadLiyaMessages()` 完成，路径固定为 `data/liyaMessages.json`。
- 图鉴 / 存档由 `loadFieldGuide()` / `saveFieldGuide()` 读写。

### 游戏状态

- 顶层持有 `gameState`，实际业务状态推进委托给 `gameSession.js`。
- `main.js` 仍直接改变一些页面状态和 UI 状态，例如 `activeOverlay`、`fieldGuideDetailCardId`、`messageView`、结算展开状态、Focus 动画状态。

### 回合推进

- `handleActionButton()` 根据按钮 `data-type` 分发到 `handleExploreAction()`、`handleDistantListenAction()`、`handleSpotSelectAction()`、`handleFirstEncounterAction()`、`handlePhotoAction()`。
- `START` 到 `START_SPOT_SELECT` / 鸟点选择由 `startGame()`、`startGameAtSpot()` 处理。
- `endGame()` 由电量 / 回合结束或 `retreat` 进入结算。

### PHOTO / FOCUS / RESULT

- `gameSession.js` 决定 PHOTO 子阶段和拍照业务结果。
- `main.js` 负责 FOCUS moving badge 的 rAF、入场 / 离场、可见行为状态捕获、对焦判定数据捕获、拍立得 overlay。
- `shoot` 前在 UI 层捕获 `capturedBehaviorState`、`capturedFocusAffix`、focus score、visual params，再调用 `handlePhotoAction()`。
- 该区域仍是高耦合，当前不建议拆。

### 消息系统入口

- 顶部工具按钮 `messages` 控制消息面板。
- `main.js` 负责消息视图切换、初始消息已读、Liya 分句动画调度、未读分隔线快照、滚动恢复状态、Liya 回复到期后的 render。
- 展示 HTML 已部分拆到 `src/ui/messagePanel.js`。

### 笔记 / 图鉴入口

- 顶部工具按钮 `fieldGuide` 控制笔记面板。
- `main.js` 负责图鉴页码、卡片详情、snapshot 翻页、发送给妹妹、自动加新 reveal、卡片 viewed 状态。
- 展示 HTML 已部分拆到 `src/ui/fieldGuidePanel.js`。

### 结算

- `renderSettlement()` / `renderSettlementSummary()` / `renderSettlementContinueAction()` 管理结算详情、展开动效、休息到下一天。
- `continueToNextDay()` 推进 `birdwatch_text_sim_day_index`，清理结算和 UI 临时态，回到 START。
- 当前结算仍与 survey flush 状态有关，但测试开关关闭时不应阻塞下一天。

### 问卷

- 开场 tester profile：`shouldShowTesterProfilePrompt()`、`renderTesterProfilePrompt()`、`submitTesterProfile()`。
- 局后 survey：`createDefaultPostSessionSurveyDraft()`、`buildPostSessionSurveyPayload()`、`submitPostSessionSurvey()`、`skipPostSessionSurvey()`、`renderPostSessionSurveySection()`、`renderSettlementSurveyEntry()`。
- 当前 `data/config.js` 的 `PLAYTEST_CONFIG` 提供 `analyticsEnabled`、`surveyEnabled`、`openingSurveyEnabled`、`settlementSurveyEnabled`、`surveyVersion`。

### analytics 调用

- `main.js` 自己维护一组 analytics runtime 计数和状态，再调用 `analytics.js` 的 `createAnalyticsSession()`、`track()`、`flush()`、`setCurrentSessionSurvey()`、`clearCurrentSessionSurvey()`。
- 主要事件：`opening_narrative_seen`、`opening_narrative_completed`、`session_start`、`session_end`、`photo_taken`、`chat_opened`、`chat_closed`、`field_guide_opened`、`sister_message_received`、`sister_message_viewed`。

### UI 渲染

- `render()` 为总入口：更新标题、状态栏、事件文本、行动按钮、详情区、日志、重置按钮、focus frame、Liya 到期 render。
- 大量 HTML string builder 仍在 `main.js`，包括结算、问卷、图鉴周边、消息周边、PHOTO detail。

### 工具函数

- DOM / class / HTML escape / 时间格式 / 稀有度展示 / 随机数 / seeded random / clamp / timestamp / scroll 等工具混在 `main.js`。
- P0 拆分可以优先从这些无业务副作用工具开始。

## 六、analytics 与 survey

### analytics 入口函数

- `src/analytics.js` 导出：`getTesterUuid()`、`getTesterProfile()`、`saveTesterProfile()`、`setCurrentSessionSurvey()`、`getCurrentSessionSurvey()`、`clearCurrentSessionSurvey()`、`createAnalyticsSession()`、`getCachedAnalyticsPayload()`、`clearCachedAnalyticsPayload()`、`getAnalyticsState()`、`track()`、`flush()`、`retryCachedAnalytics()`、`analytics` 聚合对象。
- 当前 `analytics.js` 内部有 `isAnalyticsEnabled()`，读取 `data/config.js` 的 `PLAYTEST_CONFIG.analyticsEnabled`。

### track 调用点

- `trackOpeningNarrativeSeen()`：`opening_narrative_seen`。
- `trackOpeningNarrativeCompleted()`：`opening_narrative_completed`。
- `beginAnalyticsSession()`：`session_start`。
- `finishAnalyticsSession()`：`session_end`。
- `trackPhotoTaken()`：`photo_taken`。
- `openAnalyticsChatSession()`：`chat_opened`。
- `closeAnalyticsChatSession()`：`chat_closed`。
- `trackFieldGuideOpened()`：`field_guide_opened`。
- `trackSisterMessageReceived()`：`sister_message_received`。
- `trackSisterMessageViewed()`：`sister_message_viewed`。

### flush 调用点

- `submitPostSessionSurvey()`：提交局后问卷后 `flush({ reason: "session_end", survey, finalizeSession: true })`。
- `skipPostSessionSurvey()`：跳过局后问卷后 `flush({ reason: "session_end", survey, finalizeSession: true })`。
- `flushSettlementSessionWithoutSurvey()`：休息到下一天时 `flush({ reason: "session_end", survey, finalizeSession: true })`。

### session_start / session_end 触发位置

- `session_start`：`beginAnalyticsSession(startSpotId)` 中触发；该函数由开局选点后的 action 流程调用。
- `session_end`：`finishAnalyticsSession(type, action)` 中触发；进入结算前由 action 分发流程根据 `endGame` / `retreat` 等动作收口。

### survey payload 写入位置

- payload 字段结构由 `createEmptyPostSessionSurveyAnswers()` 和 `buildPostSessionSurveyPayload()` 生成。
- 跳过问卷由 `buildSkippedPostSessionSurveyPayload()` 生成 `{ submitted: false, skipped: true, version }`。
- `setCurrentSessionSurvey(surveyPayload)` 在提交 / 跳过 / 无问卷结算 flush 前写入内存 survey。
- `savePostSurveyStatus()` 写旧状态 key `birdwatch_text_sim_post_survey_status`。
- `markDrivingSurveyCompleted()` 写当前一次性 done key `birdwatch_playtest2_driving_survey_done`。

### 测试功能开关覆盖

- `data/config.js` 当前有 `PLAYTEST_CONFIG`：`analyticsEnabled`、`surveyEnabled`、`openingSurveyEnabled`、`settlementSurveyEnabled` 默认均为 `false`。
- `src/analytics.js` 在 `createAnalyticsSession()`、`track()`、`flush()`、`retryCachedAnalytics()`、`setCurrentSessionSurvey()` 中做 analytics disabled no-op。

## 七、疑似废弃内容

### A. 可以删除

- 暂无可以直接删除项。原因：本次只读审计未做运行时覆盖验证，且多个“看似旧逻辑”的字段仍承担兼容或未来入口。

### B. 暂时只标记 deprecated

- `src/storage.js` 的 `clearFieldGuide()`：源码注释已标记 Deprecated，建议继续保留到确认无旧调用后再删。
- `birdwatch_text_sim_field_guide_v2`：仅用于 v2 到 v3 迁移，不能立刻删；可在迁移窗口结束后清理。
- `birdwatch_text_sim_post_survey_status` 与 `hasSubmittedPostSurvey()`：旧问卷状态仍有读写函数，当前疑似不再驱动 UI；建议先标记 deprecated，待确认 analytics / reset 文案是否还依赖。
- `SPOT_SELECT` 旧鸟点选择流程：`gameSession.js` 明确注释为旧流程保留给后续地图选点；当前主流程不暴露，但不能删。
- `PRECIOUS` rarity / state 相关分支：`cards.js` 注释写“未来再接”，`gameSession.js`、`rarityDisplay.js`、`sisterKnowledge.js` 中仍有兼容显示；先标记 deferred / deprecated，不要删。

- `message-editor.html` 内部浏览器内存编辑器：不是正式游戏入口，但仍是 Liya 文案生产工具，不能删；可标记 dev-only。
- `liyaMessageSystem.js` fallback 文案的 `delay.seconds = 30`：当前运行时 Liya due time 已由 fieldGuide/main 设置为 1-2 秒，fallback delay 字段暂未消费；标记为数据兼容字段。

### C. 不能碰

- `PHOTO / FOCUS / RESULT / REPOSITION / LOST` 状态机：涉及所见即所得、对焦判定、拍照消耗和 snapshot，风险高。
- `latestVisibleFocusState`、`latestFocusResult`、FOCUS rAF 和 snapshot 视觉参数：拍照正确性的核心边界。
- `liyaMessageQueueItem` 和旧 sister 字段 fallback：红点、已读、自动加新、旧存档兼容依赖。
- `data/sisterKnowledge.js`：仍是图鉴“妹妹补充”来源，不是聊天回复来源，但不能删。
- `data/liyaMessages.json`：聊天回复正式文本池，不能在清扫中顺手改文案。
- `SAVE_RESET_REGISTRY` 的 identity / infrastructure 保留组：影响 tester uuid、session index、retry payload 的 reset 行为。
- `cloudfunctions/analytics_ingest/`：线上数据链路源码，不属于前端清扫第一步。

## 八、低风险拆分候选

### P0 可立即拆

- DOM 工具：`escapeHtml()`、DOM class 切换辅助、事件目标解析、固定 DOM refs 初始化。风险低，需保持函数名和输出。
- format 工具：鸟名、时间段、rarity、计数、日期、状态文案等纯格式化函数。要求无 DOM / storage 副作用。
- random 工具：`randomBetween`、seeded random、clamp、hash 类函数。要先确认和 FOCUS 轨迹 / Liya 选择 seed 的稳定性边界。
- config helper：playtest config 读取、survey version fallback、基础配置分组。当前已有 `data/config.js` helper，可继续收口调用点。

### P1 可谨慎拆

- save manager：观鸟日、field guide load/save、reset registry、survey/tester localStorage 可以先建 facade；注意不要改 key。
- telemetry adapter：把 `main.js` 中 analytics runtime counters 和 `track*()` 聚合到独立模块；需要保证事件字段完全不变。
- data loader：Liya message load、初始消息 normalize、species/card lookup 可以拆；需避免影响首帧 fallback 和 render 刷新。

### P2 暂不拆

- PHOTO / FOCUS / RESULT：涉及动画帧、点击瞬间捕获、业务状态推进和视觉 snapshot，先写回归清单。
- 加新流程：跨 `main.js`、`fieldGuide.js`、storage、Liya 已读、自动 reveal，拆分前需测试矩阵。
- 消息队列：`liyaMessageQueueItem`、旧字段 fallback、红点、分句动画、滚动恢复耦合强。
- 回合推进：`gameSession.js` 与 `main.js` action 分发边界尚可，但当前不应在清扫第一步重排。

## 九、建议里程碑

1. 清理 debug / 废弃 CSS：只处理已确认无引用的 class 和注释，先用 `rg` 列证据；`.message-time`、旧 survey 状态先标记，不直接删。
2. 抽 utils：新增纯函数模块，迁移 escape / format / clamp / random；每次只搬一类函数，保持导出名和调用结果。
3. 抽 save manager：建立 LocalStorage key 集中读写 facade，先包 `dayIndex`、survey done、tester profile 查询，不改 key。
4. 抽 telemetry adapter：把 analytics runtime counters 从 `main.js` 迁出，保留事件字段、session_start / session_end 时机和 flush 行为。
5. 写 regression checklist：覆盖开局、拍照、对焦、结算、问卷开关、analytics 开关、Liya 多句回复、自动加新、旧存档迁移。

## 审计结论

- 当前项目入口清晰，主入口只有 `src/main.js`，但 `main.js` 职责过大。
- 当前 test / survey / analytics 开关已在 `data/config.js` 中存在，并在主要入口短路；仍建议用回归清单验证默认关闭、只开 analytics、开 analytics + survey 三种组合。
- 最低风险整理方向是先抽纯工具和文档化废弃项，暂不动 PHOTO / FOCUS / RESULT、消息队列、自动加新和回合推进。
