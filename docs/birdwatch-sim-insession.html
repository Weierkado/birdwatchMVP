2. 网格系统

2.1 墙面结构

4 面墙模拟前后左右视角，每面墙为 4×4 = 16 张卡片。玩家同时只能看到一面墙。

墙面用 绝对 ID（0–3）标识世界中的固定位置，玩家有独立的 facingWall 记录当前朝向。所有方位判断均基于相对关系计算，不使用固定语义标签。

相对方位计算方式声波位置
FRONT（当前正对）relDir === 0—（卡片直接动效）
RIGHT（玩家右侧）relDir === 1右边缘
BACK（玩家身后）relDir === 2下边缘
LEFT（玩家左侧）relDir === 3左边缘
核心公式：relDir = (bird.absoluteWall - facingWall + 4) % 4
鸟的 absoluteWall 永远不变（世界坐标）；facingWall 随玩家转身更新。 方位 HUD 每次渲染时实时计算 relDir，天然跟随朝向，无需额外同步。

2.2 墙面切换

  • 手势：左右横向滑动（swipeLeft / swipeRight）
  • 切换有过渡动画（横向滑出/滑入）
  • 拍照模式期间禁用切换手势
  • 滑动与点击的区分：位移 >30px 判定为滑动，否则判定为点击卡片

转身逻辑（facingWall 更新）：

  • 向左滑 → 向左转 → facingWall = (facingWall - 1 + 4) % 4
  • 向右滑 → 向右转 → facingWall = (facingWall + 1) % 4
滑动方向与转身方向一致（向左滑 = 向左看），符合全景照片/Street View 惯例。如实测手感反直觉，交换两条公式即可，无需改其他逻辑。

2.3 卡片状态

IDLE

鸟落入此格→ACTIVE

3. 鸟类系统

3.1 生命周期

1
入场
BirdManager 在场鸟数 < 目标值时随机选择一只鸟种入场。随机选择入场墙和入场格（排除已有鸟的格）。同时预生成 sequenceSeed。触发入场动效（卡片激活动效）。
2
网格内移动
鸟在 plannedStay 时间内可多次移动，包括跨墙移动。移动时：原格动效消失 → 目标格激活动效。同时广播 bird:moved 事件，携带 fromWall/fromIndex/toWall/toIndex。
3
鸣叫
独立于移动的随机事件。触发 AudioSystem.playCall(speciesId)。若在当前显示墙：对应格播放动效。若在其他墙：DirectionHUD.showCallEffect(direction)。同时触发 FieldGuide.markHeard(speciesId)。
4
离场
plannedStay 到期后离场。若当前正在拍照模式中，也会触发飞离（见拍照流程）。当前格动效消失，BirdManager 从活跃列表移除此鸟。

3.2 与拍照模式的关系

网格层的鸟生命周期与拍照模式内的行为序列是相互独立的两套时钟。唯一的关联点：

  • 行为序列的 FLY_AWAY 时段结束时，同步触发网格层的鸟离场
  • 网格层的 plannedStay 到期时，若鸟正在拍照模式中，行为序列提前进入 FLY_AWAY
实现建议：网格层 plannedStay 计时与行为序列总时长（5–10s）加起来构成鸟的最大存在时间。可以在入场时将 plannedStay 设为"序列时长 + 若干等待时间"，确保在序列结束时鸟自然离场，无需额外同步逻辑。

4. 拍照流程

4.1 进入条件

玩家点击处于 ACTIVE 状态的卡片 → 翻牌动画 → 动画结束时鸟仍在 → 自动进入拍照模式。

行为序列从翻牌动画结束（鸟显现）那一刻开始计时，不是从玩家按快门开始。

4.2 行为序列

序列由 sequenceSeed 驱动，伪随机生成，结构如下：

常态缓冲
随机序列段
精彩
飞离
时段时长颜色指示说明
常态缓冲固定 ~1–2s灰色序列起始的常态段，给玩家反应时间
随机序列段种子决定蓝色（有趣）/ 灰色（常态）交替有趣与常态穿插，具体时长和顺序由种子决定
精彩0.5–1 秒金/橙色，建议加脉冲动效稀有窗口，卡池最优，反应要快
飞离~0.5 秒紫色 / 淡出拍照模式结束，触发鸟离场

精彩时段 0.5–1s，无预警直接切换颜色——保留惊喜感，让玩家靠运气而非预判捕捉精彩瞬间。

4.3 卡池规则

常态时段
仅从常态卡池抽取
有趣时段
从有趣 + 常态混合卡池抽取
精彩时段
从精彩 + 有趣 + 常态全池抽取

同一只鸟的多次快门抽取相互独立，可能得到同一时段的不同卡，也可能得到重复卡。重复卡计入图鉴收集数量但不新增图鉴词条。

4.4 拍照 UI 状态

→
卡片放大
选中卡片动画放大至全屏（或接近全屏），其余 15 张卡片淡出隐藏。行为序列计时同步开始。
→
行为色彩覆层
卡片上显示当前行为时段的色彩覆层（或边框颜色）。颜色随时段切换实时变化。精彩时段加额外脉冲动效。
→
底部按钮
从屏幕下方滑入两个按钮：快门（主按钮，大）和取消拍照（次按钮，小）。
→
快门反馈
按下快门：短暂闪白（模拟拍照）→ 调用 CardDraw.draw(speciesId, currentState) → 照片存入 SDCard。卡槽数量显示更新。可连续按快门（每次消耗一个卡槽）。

4.5 拍照结束条件

结束原因触发方式结束后行为
卡槽拍满SDCard.isFull() 返回 true快门按钮禁用或隐藏，提示"存储已满"，玩家仍可取消
鸟儿飞走行为序列 FLY_AWAY 结束飞离动效 → 拍照模式自动退出
玩家取消点击取消拍照按钮直接退出拍照模式

任意结束后：卡片翻回正面朝下（IDLE），其余卡片淡入恢复显示，返回网格视图。

5. 鸟叫与方位系统

5.1 触发逻辑

鸟叫由 BirdManager 独立触发（与移动解耦）。每次鸣叫：

  1. 调用 AudioSystem.playCall(speciesId) 播放真实音频
  2. 调用 FieldGuide.markHeard(speciesId) 更新图鉴状态
  3. 判断当前显示墙：
    • 鸟在当前墙 → 对应卡片格子播放动效（抖动/声波圆圈）
    • 鸟在其他墙 → DirectionHUD.showCallEffect(direction)

5.2 方位声波 HUD

每次鸟叫，实时计算 relDir = (bird.absoluteWall - facingWall + 4) % 4，根据结果决定显示位置：

relDir含义声波位置视觉形式
0当前正对的墙—对应格子播放动效（非 HUD）
1玩家右侧屏幕右边缘向左扩散的声波弧线，2–3 次脉冲后消失
2玩家身后屏幕下边缘向上扩散的声波弧线
3玩家左侧屏幕左边缘向右扩散的声波弧线
玩家转身后，HUD 方位自动随 facingWall 重算，无需任何额外处理。声波特效持续约 1–1.5 秒后自动消失。同一时间可有多个方位特效并存（不同鸟的叫声）。拍照模式期间方位 HUD 继续正常显示。

6. UI 布局与状态

6.1 网格视图布局（主状态）

  • 顶部：倒计时 + 剩余卡槽数量
  • 中部：4×4 卡片网格（主体区域）
  • 边缘：左/右/下声波 HUD 叠层（透明浮层）
  • 墙切换：左右滑动手势，无需底部导航按钮

可选：在网格下方或顶部显示当前墙面指示（如 4 个小点，高亮当前）。

6.2 拍照视图布局

  • 中央：放大的单卡片（含行为色彩覆层）
  • 顶部：倒计时 + 剩余卡槽数（保持可见）
  • 底部滑入：快门按钮（大，居中）+ 取消按钮（小，下方或旁侧）
  • 边缘：声波 HUD 继续显示

7. 结算流程

1
局结束触发
计时器归零或所有卡槽已拍满（可选提前结算）。若在拍照模式中，强制退出拍照模式后触发。
2
照片列表读取
SDCard.getAll() 返回本局所有 PhotoData，按 capturedAt 排序。
3
逐张揭示
每张照片翻转显示卡面：卡面包含鸟种名 + 行为名、稀有度颜色、星级。可加简单揭示动效。玩家点击或自动播放切到下一张。
4
星星结算
全部揭示后显示本局总星星数（按稀有度累计）。[P2] 若接入养成系统，此处消耗星星解锁装备。
5
图鉴更新提示
本局新增的图鉴词条高亮提示（新发现的鸟种）。
